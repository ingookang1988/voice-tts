## 2. Execution Rules

1. `completed`는 실제 code path, route, test 증거가 있을 때만 사용한다.
2. `partial`은 구현은 있으나 gate나 범위가 덜 닫힌 상태다.
3. `queued`는 아직 시작하지 않았지만 다음 focus로 등록된 상태다.
4. `drift`는 코드와 목표 사이에 명시적 불일치가 남아 있는 상태다.
5. sprint는 실행 순서와 backlog만 다루며, 구조 규범은 `topology.md`, 현재 사실은 `mechanism.md`가 우선한다.
